**Zwei Befunde der Gegenprüfung:**

1. **BUG GEFUNDEN + GEFIXT (mein Code):** `bge_referenz` zeigte «**BGE BGE 87 III 100**» (doppeltes
   Präfix). Ursache: das bge-Parquet mischt die `docket_number`-Form — **20 718 / 21 228 (~98 %)**

   `87-III-100`), nur die Anzeige.
2. **voilaj-Datenqualität (Quell-Quirk, dokumentiert, kein Blocker):** für einige ältere bger ist
   das Feld `decision_date` NICHT das Urteilsdatum. Bsp. 1P.696/2000: Parquet `decision_date`
   2000-05-29, aber `publication_date` 2001-01-03 = JumpCGI-id = Urteilstext «3. Januar 2001». Wir
   mappen `decision_date` → `datum` **quelltreu**; die Abweichung ist eine voilaj-Semantik (E4/später:
   ggf. Rekonziliation via publication_date/Text). §8-markiert, `kuratierung='maschinell'` = ungeprüfte
   Masse.

## 9. Bereit für VPS-Upload — Checkliste (Serving-Schritt, David bestellt separat)

1. **Datei-rsync:** `daten/masse.db` (~5,8 GB) auf den VPS (≥ 350 GB NVMe / ≥ 32 GB RAM, §6.3).
2. **cold-FTS bauen (auf dem VPS, NICHT lokal — 58-GB-Klasse, transiente Rebuild-Spitze ~200 GB):**
   `fts_entscheide_masse` als FTS5 external-content über `entscheide` (Text = `full_text`), Tokenizer
   `unicode61 remove_diacritics 2` (wie hot-FTS). Kommando-Skizze:
   ```sql
   CREATE VIRTUAL TABLE fts_entscheide_masse USING fts5(
     full_text, content='entscheide', content_rowid='rowid',
     tokenize='unicode61 remove_diacritics 2');
   INSERT INTO fts_entscheide_masse(rowid, full_text) SELECT rowid, full_text FROM entscheide;
   ```
   (RAM-Reserve ≥ 32 GB, sonst abgebrochener Merge — §6.3.)
3. **Read-API:** schlanke read-only Query-Schicht (libSQL-Server oder Read-API), bm25 + snippet,
   Pagination by design (< 4,5-MB-Payload), NIE Volltext in der Trefferliste — analog dem hot-Pfad
   `scripts/datenhaltung/suche.ts`.
4. **Long-Tail-Route `/rechtsprechung/:key`:** on-demand-Edge-Fetch für nicht-prerenderte Keys,
   inhaltsvollständig (§15.6: voller Text im DOM/Ctrl+F, `#`-Anker, Provenienz §7 a–d, amtlicher
   Live-Link als Fallback); DIESELBE Reader-Komponente wie das Schaufenster (§11.1).
5. **resolve-zitate nach jedem Reingest** (idempotent) + **`norm_rangliste`-Rebuild = E4** (hier
   bewusst NICHT gebaut).
6. **VPS-Angebot** (E3-Vorbedingung 2, §6.3) — reales Anbieter-Angebot gegen die Posten-Tabelle.
